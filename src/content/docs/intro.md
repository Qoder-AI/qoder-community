## Quest 1.0 产品概述

Quest 源自拉丁语 quaerere（to seek, to inquire），意为"有目的的探索"。

### Quest 不是 Task

- **Task** 是被分配的事务，强调"完成"
- **Quest** 是自主的探索，强调"过程"与"目标"

在 Quest Mode（任务探索模式）下，agent 会自主探索，实现你定义的任务。

## 一句话定位

> Quest 是全球首个可实时自我学习与进化的自主编程智能体。你不需要告诉它"怎么做"，只需要告诉它"要什么"。它会自己探索、自己学习、自己执行、自己验证，最后把结果交给你。

**人定义目标，Quest 自主完成**

## 产品愿景

让每个人都拥有一个 Agent 工程师团队。

> **Quest on, Hands off** — 委派任务后放手，AI 自主完成。

## 自进化能力

Quest 1.0 不仅是 Autonomous Agent（自主智能体），更是 Self-Evolving Agent（自进化智能体）。

### 连续学习（Continuous Learning）
- 深度分析项目代码库结构、架构演进、历史设计决策
- 自动生成项目知识图谱，理解而非仅仅执行
- 积累可复用的 SPEC 模板

### 实时自我反思（Real-time Self-Reflection）
- 执行过程中遇到错误自动诊断、自主修复
- 采用"生成-验证-修复"闭环，每一次试错都提升决策质量
- 遇错自修，不打扰用户

### 知识内化（Knowledge Internalization）
- 从显式的 SPEC 文档到隐式的决策直觉
- 跨任务知识迁移，越用越懂你的项目
- 多子 Agent 并行探索，策略持续优化

## 核心设计原则

Quest 的设计围绕五大核心原则：

### 1. 可控性（Control）
- 开发者始终掌控全局，像管理团队一样管理 AI 任务
- 多层监控实时追踪进度
- 支持随时中断、干预、恢复

### 2. 自主性（Autonomy）
- AI 可以长时间自主工作，无需持续人工参与
- 云端任务支持 24 小时不间断执行
- 从需求理解到代码实现到测试验证，全流程自主完成

### 3. 协同性（Collaboration）
- 多个 AI 并行工作，像团队一样协同
- 三种执行环境灵活选择
- 独立沙箱互不干扰

### 4. 可验证（Verifiability）
- SPEC 文档驱动保证任务质量
- 产物透明可追溯
- 支持 Review 后迭代优化

### 5. 深度理解（Deep Understanding）
- 基于项目历史和架构演进理解代码
- 通过 Repo Wiki 理解代码库结构
- 结合历史知识生成符合项目特点的方案

## 目标用户

### A 类：专业开发者
- 有 80-100% 信心自主完成开发
- 现在使用：Cursor / Copilot / Claude Code
- Quest 价值：委派复杂任务，异步交付

### B 类：半熟练开发
- 有 20-80% 信心自主完成开发
- 现在使用：Lovable / Replit / Claude Chat
- Quest 价值：探索性编程，边学边做

### C 类：非开发背景
- 有 0-20% 信心自主完成开发
- 现在使用：Lovable / Manus / 豆包
- Quest 价值：想法直接变现，无需懂技术

## 与竞品的差异

| 维度 | Chat/Agent 模式 | **Quest 模式** |
|------|----------------|---------------|
| **核心理念** | 人机协作，人参与度高 | 自主执行，人参与度低 |
| **执行过程** | 展示细节，用户跟随 | 折叠过程，关注产物 |
| **中断策略** | 遇到不确定就问用户 | 先尝试自行解决 |
| **效果上限** | 人+AI 协作上限更高 | 追求自主上限，释放人力 |
| **目标用户** | 愿意深度参与的开发者 | 希望委派任务的用户 |
