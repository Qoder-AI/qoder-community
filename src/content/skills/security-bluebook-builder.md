## Use Cases

- Application security architecture design
- Threat modeling analysis
- Security compliance documentation
- Data classification strategy
- Security audit preparation

## Core Capabilities

- **Threat Modeling**: STRIDE and other methodologies
- **Data Classification**: Sensitivity level categorization
- **Authentication Design**: Secure identity verification
- **Audit Logging**: Compliant log recording

## Example

```
Please help build a security bluebook for a payment system:

System features:
- User payments
- Subscription management
- Billing history

Include:
1. Threat model (STRIDE analysis)
2. Data classification (PII, payment data, etc.)
3. Authentication and authorization design
4. Audit logging requirements
5. Security checklist
```

## Notes

- Customize based on industry standards
- Regularly update threat assessments
- Collaborate with compliance teams
- Keep documentation synced with implementation
