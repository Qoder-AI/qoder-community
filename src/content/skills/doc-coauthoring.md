## Use Cases

- Write technical docs and API documentation
- Create product requirement documents (PRD)
- Draft proposals and decision docs
- Collaborative long-form content
- Structured document iteration process

## Core Capabilities

- **Context Transfer**: Efficiently convey writing background and requirements
- **Structured Iteration**: Optimize content through multiple feedback rounds
- **Reader Verification**: Ensure document works for target readers
- **Version Management**: Track document change history

## Example

```
Let's co-author a technical spec document:

Topic: User authentication system refactoring
Target readers: Backend development team
Document structure:
1. Background and goals
2. Current issues
3. Solution design
4. Implementation plan
5. Risk assessment

Please help complete the first draft of section one.
```

## Notes

- Suitable for complex documents requiring multiple iterations
- Clearly define target readers
- Maintain document structure consistency
