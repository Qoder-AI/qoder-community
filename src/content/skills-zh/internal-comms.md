## 使用场景

- 周报和项目状态更新
- 3P 更新（进展、计划、问题）
- 全公司公告
- 事故报告和复盘
- 团队通讯和文化内容

## 核心能力

- **模板化写作**：使用公司标准格式
- **信息提炼**：将复杂信息简化为清晰要点
- **受众适配**：根据不同读者调整语气和详细程度
- **一致性**：保持统一的沟通风格

## 示例

```
请帮我写这周的 3P 更新：

进展：
- 完成用户认证模块重构
- 性能优化提升 30%

计划：
- 下周开始支付系统集成
- 准备技术评审会议

问题：
- 第三方 API 响应延迟
- 需要额外的测试资源
```

## 注意事项

- 根据公司文化调整语气
- 保持简洁，突出重点
- 妥善处理敏感信息
