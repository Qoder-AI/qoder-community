## 使用场景

- 规范化 Git 提交信息
- 自动创建 Pull Request
- 代码审查流程管理
- 分支管理策略实施
- 发布流程自动化

## 核心能力

- **Conventional Commits**：符合规范的提交信息
- **PR 模板**：标准化的 PR 描述
- **审查清单**：确保代码质量
- **分支命名**：统一的分支命名规范

## 示例

```
我已完成用户认证功能开发，请帮我：
1. 生成符合 Conventional Commits 的提交信息
2. 创建包含变更描述和测试步骤的 PR
3. 列出审查时需要关注的要点
```

## 注意事项

- 每次提交只做一件事
- 保持 PR 大小合理
- 清晰描述为什么做这些修改
- 包含测试验证步骤
