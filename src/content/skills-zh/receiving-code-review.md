## 使用场景

- 处理 PR 审查评论
- 理解和回应反馈
- 决定是否采纳建议
- 与审查者有效沟通
- 将反馈转化为改进

## 核心能力

- **积极态度**：以学习心态接受反馈
- **理解意图**：确保理解审查者意图
- **有效回应**：清晰回应每条评论
- **实施改进**：将反馈转化为代码改进

## 示例

```
我收到了以下代码审查评论：
1. "这个函数太长了，建议拆分"
2. "错误处理可以更健壮"
3. "考虑使用策略模式"

请帮我分析这些反馈并制定改进计划。
```

## 注意事项

- 对事不对人
- 不懂就问，不要猜测
- 感谢审查者的时间
- 保持专业和礼貌
