## 使用场景

- 编写技术文档和 API 文档
- 创建产品需求文档 (PRD)
- 起草提案和决策文档
- 长篇内容协作
- 结构化文档迭代流程

## 核心能力

- **上下文传递**：高效传达写作背景和需求
- **结构化迭代**：通过多轮反馈优化内容
- **读者验证**：确保文档对目标读者有效
- **版本管理**：跟踪文档变更历史

## 示例

```
让我们一起协作编写一份技术规范文档：

主题：用户认证系统重构
目标读者：后端开发团队
文档结构：
1. 背景和目标
2. 现有问题
3. 解决方案设计
4. 实施计划
5. 风险评估

请先帮我完成第一部分的初稿。
```

## 注意事项

- 适合需要多次迭代的复杂文档
- 明确定义目标读者
- 保持文档结构一致性
