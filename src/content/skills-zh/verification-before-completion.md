## 使用场景

- 功能完成前的自检
- 代码提交前的最终检查
- 发布前的质量验证
- 交付物完整性确认
- 减少返工和遗漏

## 核心能力

- **检查清单**：系统化的验证项目
- **自动检查**：可自动验证的项目
- **人工验证**：需要人工确认的项目
- **文档完整性**：检查相关文档是否更新

## 示例

```
我已完成订单模块的重构，请帮我进行完成前验证：
1. 功能完整性检查
2. 测试覆盖检查
3. 代码质量检查
4. 文档更新检查
5. 性能影响评估
```

## 注意事项

- 根据项目类型定制检查清单
- 不要跳过任何检查项
- 记录验证结果
- 及时修复发现的问题
