## 使用场景

- 环境变量安全管理
- 密钥轮换
- 配置安全审计
- 防止敏感信息泄露
- 多环境配置管理

## 核心能力

- **泄露检测**：识别代码中的硬编码密钥
- **安全存储**：推荐安全存储方案
- **访问控制**：最小权限原则
- **审计追踪**：密钥使用记录

## 示例

```
请审计项目的环境变量安全性：

检查：
1. 代码中是否有硬编码的密钥
2. .env 文件是否在 .gitignore 中
3. 环境变量命名是否规范
4. 敏感变量是否有适当的访问控制

项目路径：./src
```

## 注意事项

- 永远不要将密钥提交到代码仓库
- 使用专门的密钥管理服务
- 定期轮换敏感凭证
- 限制密钥的访问范围
